# Spec Flow

Run the Spec Flow: research a spec, RFC, or design issue, then produce the plan/design artifact and stop before implementation.

## Workflow

1. `research` — gather context and clarify the design problem.
2. `plan` — produce the spec or implementation plan artifact.

## Output

A plan or design artifact linked to the source issue.

## Restrictions

- Do not open an implementation PR.
- Do not modify product code.
