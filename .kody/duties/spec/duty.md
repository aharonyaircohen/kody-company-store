# Issue Specification

## Job

Turn an unclear issue into a usable implementation specification.

## Executable

Run the `spec` executable. The engine owns the implementation details.

## Output

A clearer issue spec with acceptance criteria or blocking questions.

## Allowed Commands

- Run the `spec` executable.

## Restrictions

- Do not implement code.
- Ask blocking questions when requirements are missing.
- Keep the spec tied to the original issue.
