# Docs Health - catch documentation drift and coverage gaps

## Job

Find documentation drift from merged PRs and important source areas that lack useful in-code documentation.

## Executable

Run the `docs-health` executable. Its skills own the README drift and in-code coverage methods.

## Output

Tracking issues and inbox recommendations for new documentation drift or the worst documentation coverage gap.

## Allowed Commands

- Run the `docs-health` executable.

## Restrictions

- Advisory only.
- Do not edit code or docs directly.
- Keep the existing per-skill deduplication rules.
- Do not open PRs, merge, approve, or push branches.
